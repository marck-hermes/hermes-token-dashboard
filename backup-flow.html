🎯 Cenários de Disaster Recovery

💥 Servidor Perdido / VM Corrompida

1. Provisionar novo servidor Ubuntu 24.04 ARM
2. curl -fsSL hermes-agent... | bash
3. rclone copy gdrive:Hermes/backups/hermes-backup-YYYY-MM-DD.zip .
4. hermes import hermes-backup-YYYY-MM-DD.zip
5. hermes status → validar

🔧 Config Quebrada / Rollback

1. cd ~/github/hermes-config && git log --oneline -10
2. git checkout HEAD~1 -- config.yaml profiles/str/config.yaml
3. hermes-config-save "revert: rollback config"
4. Se precisa secrets: baixar Layer 3 do GDrive

🔑 Secret Vazado / Rotação de Credencial

1. Revogar credencial antiga no provedor (OpenRouter, Vercel, GitHub, Telegram)
2. Gerar nova credencial
3. Atualizar ~/.hermes/profiles/*/.env
4. hermes-config-save-and-backup "security: rotate OpenRouter key"
5. Verificar no GDrive: rclone lsf gdrive:Hermes/backups/configs-non-sanitized/

🦠 Ransomware / Corrupção Silenciosa

1. Identificar data do último backup limpo (12 meses histórico)
2. Baixar Layer 1 da data: rclone copy gdrive:Hermes/backups/hermes-backup-YYYY-MM-DD.zip .
3. hermes import hermes-backup-YYYY-MM-DD.zip
4. Validar hermes status + testar pipeline STR

Backup & Restore

# Backup manual completo
hermes backup -o ~/hermes-backup-$(date +%F-%H%M).zip

# Backup rápido (só estado crítico)
hermes backup --quick --label "antes-do-update"

# Script mensal completo (full + sanitized)
~/scripts/hermes-backup-monthly.sh

# Backup NÃO-sanitizado (COM secrets) - MANUAL
~/scripts/hermes-backup-non-sanitized.sh
# ou via cron job pausado
hermes cron run ccd336ac03f8

GitHub Configs

# Commit + push rápido (mudança trivial)
hermes-config-save "docs: fix typo"

# Commit + push + backup NÃO-sanitizado (mudança de secret)
hermes-config-save-and-backup "feat: add new model for Nexo"

# Ver status
cd ~/github/hermes-config && git status
